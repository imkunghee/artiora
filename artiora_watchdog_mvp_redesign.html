The Problem

AI 시대의 문제는
“그림 생성”이 아니라 출처 증명입니다.

공개된 작품은 무단 학습 데이터로 흡수될 수 있고, 복제된 화풍이 유통되어도 원작자에게 보상과 증거가 남지 않는 문제가 있습니다. MVP는 이 문제를 작은 단위의 작동 데모로 줄입니다.

01

원본 도용

작가의 공개 이미지가 수집·변형·재배포되어도 원본과 연결되는 증거가 약합니다.

02

신원 부재

“실제 그 작가가 승인했는가”를 확인하는 인증 흐름이 아직 표준화되어 있지 않습니다.

03

수익 단절

화풍이 상업적으로 쓰여도 작가 승인과 정산 구조가 분리되어 있으면 보상이 흐르기 어렵습니다.

Core Architecture

MVP 이후 확장 구조

현재 파일은 “작동 증명”에 집중합니다. 실제 서비스화 단계에서는 DID 인증, 원본 해시 앵커링, Watchdog 탐지, Style API 정산으로 연결됩니다.

01 IDENTITY작가 신원 확인

모바일 신분증 또는 DID 기반으로 본인=작가=지갑 일치성을 검증합니다.

02 ASSET원본 서명

작품 ID, 작가 식별자, 사용 정책을 원본 이미지와 연결합니다.

03 PROTECTION노출 감시

재업로드·무단 사용·학습 위험 의심 케이스를 리포트화합니다.

04 LICENSE승인 기반 수익화

Style API 또는 라이선스 계약으로 작가 승인 사용만 정산합니다.
